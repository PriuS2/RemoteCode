# OpenCode Web 통합 구현 플랜

## 개요

OpenCode Web 타입 세션을 추가하여, 이 프로젝트의 앱 내부(iframe)에서 OpenCode Web 인터페이스를 표시한다.

---

## 배경

- 현재 세션 타입: ClaudeCode, OpenCode, Terminal, Custom
- 목표: OpenCode Web 타입 추가
- 동작: 세션 열 때 OpenCode Web 서버가 실행중이지 않으면 서버를 시작하고, 앱 내부 iframe으로 표시
- 외부 접근: CloudFlare Tunnel으로 이미 8000번 포트 노출 중, 프록시 방식으로 OpenCode Web도同一 포트로 접근 가능하게 함

---

## 구현 항목

### 1. 백엔드 변경

#### 1.1 설정 추가 (`backend/config.py`)

```python
opencode_web_port: int = 8096  # OpenCode Web 서버 포트
opencode_web_hostname: str = "0.0.0.0"  # 모든 인터페이스에서 수신
```

#### 1.2 OpenCode Web 서버 매니저 (`backend/opencode_web_manager.py` - 新規)

```python
class OpenCodeWebManager:
    def __init__(self):
        self._process: subprocess.Popen | None = None
        self._port: int | None = None

    def start(self) -> int:
        """서버 시작, 할당된 포트 반환"""

    def stop(self) -> None:
        """서버 중지"""

    def get_status(self) -> dict:
        """상태 반환: running, port"""

    def is_running(self) -> bool:
        """실행 여부 확인"""

opencode_web_manager = OpenCodeWebManager()
```

**동작:**
- `opencode web --port <dynamic> --hostname 0.0.0.0` 로 서버 시작
- 랜덤 포트 사용 시实际 포트 번호를 파일/변수로 기록
- 브라우저 자동 열기 방지: `opencode web`은 기본 브라우저를 열기 때문에 별도 처리 필요 (환경 변수 또는 프로세스层面)

#### 1.3 API 엔드포인트 추가 (`backend/main.py`)

| 엔드포인트 | 방법 | 설명 |
|-----------|------|------|
| `/api/opencode-web/status` | GET | 서버 상태 확인 |
| `/api/opencode-web/start` | POST | 서버 시작 |
| `/api/opencode-web/stop` | POST | 서버 중지 |
| `/api/opencode-web/proxy/{path:path}` | GET, POST, PUT, DELETE | HTTP 프록시 |
| `/api/opencode-web/ws/{path:path}` | WebSocket | WebSocket 프록시 (필요시) |

**프록시 동작:**
- 클라이언트 요청을 `http://localhost:<port>`로 전달
- 응답을 클라이언트에 반환
- 인증: 이 프로젝트 로그인 토큰 검증 (프록시 통과 전)

#### 1.4 세션 관리 수정 (`backend/session_manager.py`)

- `cli_type === "opencode-web"` 인 경우 PTY 생성하지 않음
- work_path는 단순 표시용으로 저장 (OpenCode Web에서 사용 안 함)

#### 1.5 데이터베이스 마이그레이션

- 기존 DB에 `cli_type` 컬럼이 있으면 새 값("opencode-web") 허용 확인

```tsx
interface OpenCodeWebViewerProps {
  token: string;
  onClose: () => void;
}

export default function OpenCodeWebViewer({ token, onClose }: OpenCodeWebViewerProps) {
  const [status, setStatus] = useState<string>("loading");
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    // 1. 상태 확인
    fetch("/api/opencode-web/status", { headers: authHeaders() })
      .then(res => res.json())
      .then(data => {
        if (!data.running) {
          // 2. 미실행 시 시작
          return fetch("/api/opencode-web/start", { method: "POST", headers: authHeaders() });
        }
      })
      .then(() => setStatus("ready"))
      .catch(err => setError(err.message));
  }, []);

  if (status === "loading") return <div>Loading...</div>;
  if (error) return <div>Error: {error}</div>;

  return (
    <iframe
      src="/api/opencode-web/proxy/"
      style={{ width: "100%", height: "100%", border: "none" }}
    />
  );
}
```

#### 2.5 메인 앱 수정 (`frontend/src/App.tsx`)


## 동작 흐름

```
1. 사용자: 새 세션 → "OpenCode Web" 선택 → 생성
2. 백엔드: cli_type="opencode-web"로 세션 생성 (PTY 미생성)
3. 프론트엔드: 세션 선택 → OpenCodeWebViewer 표시
4. OpenCodeWebViewer:
   a. /api/opencode-web/status 확인
   b. 미실행 시 /api/opencode-web/start 호출하여 서버 시작
   c. iframe으로 /api/opencode-web/proxy/ 표시
5. 외부 접근: Tunnel 8000번 포트로 /api/opencode-web/* 요청 시 프록시 전달
```

## 고려사항

### 1. 브라우저 자동 열기 방지

`opencode web`은 기본적으로 브라우저를 자동 열음. 옵션으로 끌 수 없다면:
- Windows: 레지스트리 또는 프로세스 시작 후 창 최소화
- 별도 브라우저 열림 방지 프로세스 구현 필요

### 2. WebSocket 프록시

OpenCode Web이 WebSocket을 사용하는 경우:
- `/api/opencode-web/ws/{path}` WebSocket 엔드포인트 추가
- 클라이언트 WebSocket을 백엔드 WebSocket으로 브릿지

### 3. work_path 의미 재정의

- 기존 타입: work_path에서 CLI 실행
- opencode-web: work_path는 단순 표시용 (OpenCode Web에서 직접 사용 안 함)

### 4. 인증

- 프록시 요청 시 이 프로젝트 JWT 토큰 검증
- OpenCode Web 자체 인증은 사용 안 함 (프록시에서 인증 처리)
