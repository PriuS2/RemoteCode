# 개발개요

> 이 문서는 Claude Code plan 모드에서 구현 계획을 설계하기 위한 프로젝트 상세 개요입니다.
> 작성일: 2026-02-25 | 최종 수정일: 2026-02-25

---

## 1. 프로젝트 정의

- **프로젝트명:** Claude Code Remote
- **목적 및 배경:** 외부에서 웹 브라우저를 통해 로컬 Windows 11 환경의 Claude Code CLI를 원격으로 사용할 수 있게 하는 서비스. 로컬에서 직접 CLI를 사용하는 것과 동일한 경험을 제공한다.
- **대상 사용자:** 단일 사용자 (본인)
- **핵심 가치:** 장소에 구애받지 않고 웹 브라우저만으로 로컬 Claude Code 환경에 접근. 세션별 작업 경로 관리 및 세션 복원(resume) 지원.

## 2. 기술 스택

| 구분 | 기술 | 선택 이유 |
|------|------|-----------|
| 프론트엔드 | React + xterm.js + Vite | xterm.js는 웹 터미널 에뮬레이션 표준. 세션 관리 UI에 React 컴포넌트 구조 적합. Vite로 빌드 |
| 백엔드 | Python (FastAPI) | WebSocket 네이티브 지원, 비동기 처리 우수, subprocess 관리 용이 |
| 데이터베이스 | SQLite (파일 기반) | 단일 사용자 환경에 적합. 별도 DB 서버 불필요. 세션 메타정보 저장용 |
| PTY | pywinpty | Windows 환경에서 pseudo-terminal 처리. ANSI 제어 코드, 색상, 커서 이동 등 완전한 터미널 에뮬레이션 지원 |
| 통신 | WebSocket | 터미널 실시간 양방향 통신에 필수 |
| 배포 | FastAPI 통합 서빙 | 프론트엔드 빌드 결과물을 FastAPI가 정적 파일로 서빙. 단일 서버, 단일 포트 |
| 인프라 | 로컬 Windows 11 | 기본: 포트포워딩 / 대안: Cloudflare Tunnel |

## 3. 기능 요구사항

### 3.1 핵심 기능 (MVP)

- **웹 터미널:** xterm.js 기반 터미널 에뮬레이션. Claude Code CLI와 동일한 입출력 경험 (색상, ANSI 제어 코드 완전 지원)
- **세션 생성:**
  - 원하는 로컬 경로를 지정하여 새 Claude Code 세션 생성
  - 경로가 존재하지 않을 경우 폴더 자동 생성 옵션 제공
  - 다른 세션이 실행 중인 상태에서도 새 세션 생성 가능
- **세션 목록/전환:**
  - 세션 목록 조회 (active / suspended / closed 상태 구분)
  - 세션 간 자유롭게 전환 (활성 세션의 프로세스는 백그라운드에서 계속 실행)
  - 실행 중인 세션에서 다른 세션으로 전환해도 기존 세션 유지
- **세션 제어 (3가지 종료 방식):**
  - **전환 (Switch):** 현재 세션을 그대로 실행 상태로 두고 다른 세션으로 이동. 프로세스 유지.
  - **일시중단 (Suspend):** PTY에 `/exit` 명령을 전송하여 Claude Code를 정상 종료. 세션 정보(claude_session_id)를 보존하여 나중에 `--resume`으로 대화 이어가기 가능. 상태: suspended
  - **완전 종료 (Terminate):** Claude Code 프로세스를 종료하고 세션을 닫음. resume 불가. 상태: closed
- **세션 복원 (Resume):**
  - suspended 상태의 세션을 `claude --resume` 플래그를 활용하여 이전 대화를 이어서 재개
  - 서버 재시작 후에도 세션 목록(경로, claude_session_id) 유지, suspended 세션 resume 가능
  - 서버 재시작 시 기존 active 세션은 자동으로 suspended로 전환 (프로세스가 이미 종료되었으므로)
- **세션 정리:**
  - closed 세션은 사용자가 목록에서 수동 삭제
- **인증:** 서버 시작 시 설정한 비밀번호로 로그인

### 3.2 추가 기능 (확장)

- 파일 탐색기 (작업 경로의 파일 구조 확인)
- 다크/라이트 테마 전환
- 모바일 반응형 UI
- Cloudflare Tunnel 연동 가이드/자동 설정

## 4. 비기능 요구사항

- **보안:** 비밀번호 인증 필수. 외부 노출 시 HTTPS 권장
- **성능:** 터미널 입출력 지연 최소화 (WebSocket 기반 실시간 통신)
- **안정성:** Claude Code 프로세스 비정상 종료 감지 시 세션 상태를 자동 반영. 백그라운드에서 실행 중인 세션 프로세스의 헬스체크

## 5. 시스템 아키텍처

```
[웹 브라우저]
    │
    │  HTTPS + WebSocket
    │
    ├── (방법 1) 포트포워딩으로 직접 접속
    │
    ├── (방법 2) Cloudflare Tunnel 경유
    │       └── cloudflared 데몬이 로컬에서 Cloudflare로 아웃바운드 연결
    │           → 인바운드 포트 오픈 불필요, HTTPS 자동 적용
    │
    ▼
[FastAPI 서버 (Windows 11 로컬)]
    ├── REST API: 인증, 세션 CRUD
    ├── WebSocket: 터미널 입출력 스트리밍
    ├── 프로세스 매니저: pywinpty로 Claude Code CLI PTY 프로세스 관리
    └── SQLite: 세션 메타 정보 저장
         │
         ▼
[Claude Code CLI 프로세스] × N개 (세션당 1개)
    ├── 각 세션은 지정된 작업 경로(cwd)에서 실행
    └── resume 시: claude --resume <session_id> 로 이전 대화 복원
```

## 6. 데이터 모델

### sessions 테이블

| 필드 | 타입 | 설명 |
|------|------|------|
| id | TEXT (UUID) | 세션 고유 식별자 (서버 내부용) |
| claude_session_id | TEXT | Claude Code의 세션 ID (resume에 사용) |
| name | TEXT | 세션 표시명 (선택, 기본값: 작업 경로 폴더명) |
| work_path | TEXT | Claude Code 작업 경로 |
| created_at | DATETIME | 세션 생성 시각 |
| last_accessed_at | DATETIME | 마지막 접근 시각 |
| status | TEXT | active / suspended / closed |

## 7. API 설계

### REST API

| 메서드 | 경로 | 설명 |
|--------|------|------|
| POST | /api/auth/login | 비밀번호 인증 |
| GET | /api/sessions | 세션 목록 조회 |
| POST | /api/sessions | 새 세션 생성 (작업 경로 지정, 폴더 생성 옵션) |
| POST | /api/sessions/{id}/resume | suspended 세션을 resume으로 재개 |
| POST | /api/sessions/{id}/suspend | 세션 일시중단 (프로세스 정상 종료, resume 가능) |
| DELETE | /api/sessions/{id} | 세션 완전 종료 (프로세스 강제 종료, resume 불가) |

### WebSocket

| 경로 | 설명 |
|------|------|
| /ws/terminal/{session_id} | 터미널 입출력 스트리밍 (PTY ↔ xterm.js) |

### WebSocket 메시지 프로토콜 (JSON)

| 방향 | type | data | 설명 |
|------|------|------|------|
| Client → Server | input | string | 사용자 키 입력 |
| Client → Server | resize | {cols, rows} | 터미널 크기 변경 |
| Server → Client | output | string | PTY 출력 데이터 |
| Server → Client | status | string | 세션 상태 변경 알림 (suspended, closed 등) |

## 9. 제약 조건 및 고려사항

- **OS 종속성:** 로컬 환경이 Windows 11이므로 PTY 처리에 pywinpty 사용. Linux/macOS 호환은 고려하지 않음
- **Claude Code CLI 의존성:** 로컬에 Claude Code CLI가 설치 및 인증되어 있어야 함
- **PTY 처리:** pywinpty를 통해 완전한 터미널 에뮬레이션 구현. ANSI 색상, 커서 이동, 화면 크기 조절(resize) 모두 지원해야 함
- **동시 세션 수:** Claude Code API 동시 호출 제한 확인 필요. 과도한 동시 세션은 API rate limit에 걸릴 수 있음
- **외부 접속 방법:**
  - **기본:** 포트포워딩 (라우터에서 특정 포트를 로컬 서버로 매핑)
  - **대안:** Cloudflare Tunnel (포트포워딩 불가 환경용. `cloudflared` 설치 후 아웃바운드 터널 생성. 인바운드 포트 오픈 불필요, HTTPS 자동 적용)
- **세션 resume:** Claude Code의 `--resume` 플래그는 Claude Code 자체 세션 ID를 사용. 서버에서 이 ID를 추적하여 저장해야 함
- **서버 재시작 복구:** 서버 시작 시 DB의 active 세션을 일괄 suspended로 전환하는 초기화 로직 필요
